How This Works

1. Generate

Pick a domain and template. Files are generated locally in output/ with placeholders replaced.

2. Review

Preview the generated site right in the portal. Confirm layout, links, and content before deployment.

3. Deploy

{% if auto_push %}

GitHub auto-push is enabled. A repo push and Pages configuration are attempted automatically after generation.

{% else %}

GitHub auto-push is disabled. Generate locally, review, then push when you are ready.

{% endif %}

4. Verify Reputation

Use Reputation Check and Domain Finder to validate WHOIS history, reputation feeds, and categorization health.
